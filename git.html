Git'e Başlangıç

Git, yazılım geliştirme sürecinde dosyaların değişikliklerini takip eden dağıtık bir versiyon kontrol sistemidir.

Git Kurulumu

# Windows için
# https://git-scm.com/ adresinden indirin

# macOS için
brew install git

# Linux (Debian/Ubuntu) için
sudo apt install git

İlk Yapılandırma

# Kullanıcı bilgilerini ayarlama
git config --global user.name "Adınız Soyadınız"
git config --global user.email "email@adresiniz.com"

# Ayarları kontrol etme
git config --list

Temel Git Komutları

Günlük kullanımda en sık ihtiyaç duyacağınız Git komutları

git init
Yeni bir Git deposu oluşturur
git clone [url]
Uzak depoyu yereline kopyalar
git status
Dosya durumlarını gösterir
git add [file]
Dosyaları staging area'ya ekler
git commit -m "mesaj"
Değişiklikleri kaydeder
git log
Commit geçmişini gösterir

Günlük Çalışma Akışı

1
Değişiklikleri Kontrol Et
git status
2
Değişiklikleri Staging'e Ekle
# Tüm değişiklikleri ekle
git add .

# Belirli dosyaları ekle
git add dosya1.js dosya2.css
3
Değişiklikleri Commit Et
git commit -m "Kullanıcı arayüzü güncellemeleri"

Branch (Dal) Yönetimi

Branch'ler, projenizin farklı versiyonlarını yönetmenizi sağlar.

Temel Branch Komutları

# Branch listesini görüntüle
git branch

# Yeni branch oluştur
git branch yeni-ozellik

# Branch değiştir
git checkout yeni-ozellik

# Branch oluştur ve geçiş yap (kısa yol)
git checkout -b yeni-ozellik

Branch Görselleştirme

main* (initial commit)
feature/login (branch from main)
feature/payment (branch from main)

Merge (Birleştirme) İşlemleri

# Önce main branch'ine geç
git checkout main

# Yeni özellik branch'ini main'e birleştir
git merge yeni-ozellik

Uzak Depo (Remote) İşlemleri

GitHub, GitLab veya Bitbucket gibi platformlarla çalışma

Remote Komutları

# Uzak depo ekleme
git remote add origin https://github.com/kullanici/repo.git

# Uzak depoya gönderme (push)
git push -u origin main

# Uzak depodan çekme (pull)
git pull origin main

# Uzak depoyu görüntüle
git remote -v

Ekip Çalışması Akışı

1
Son Değişiklikleri Al
git pull origin main
2
Değişiklikleri Yap ve Commit Et
git add .
git commit -m "Yeni özellik eklendi"
3
Değişiklikleri Gönder
git push origin main

İleri Seviye Git Komutları

Daha gelişmiş senaryolar için kullanışlı Git komutları

Değişiklikleri Geri Alma

# Son commit'i geri al (soft - değişiklikler kalır)
git reset --soft HEAD~1

# Son commit'i geri al (hard - değişiklikler silinir)
git reset --hard HEAD~1

# Belirli bir dosyayı geri al
git checkout -- dosyaadi.js

Stash (Geçici Saklama)

# Geçerli değişiklikleri geçici olarak sakla
git stash

# Saklanan değişiklikleri listele
git stash list

# Son saklanan değişiklikleri geri yükle
git stash pop

Hata Ayıklama ve Log

# Detaylı log görüntüleme
git log --oneline --graph --all

# Belirli bir dosyanın geçmişi
git log -p dosyaadi.js

# Kim, ne zaman, hangi dosyayı değiştirdi
git blame dosyaadi.js

En İyi Uygulamalar

# Commit mesajları anlamlı olsun
"feat: kullanıcı giriş sayfası eklendi"
"fix: güvenlik açığı kapatıldı"
"docs: readme güncellendi"
  • Küçük ve sık commit yapın
  • Branch'lerde çalışın
  • Commit'lerden önce test edin
  • Anlamlı commit mesajları yazın

Sık Yapılan Hatalar

  • Doğrudan main branch'inde çalışmak
  • Commit mesajı olarak "update" yazmak
  • Pull yapmadan push denemek
  • .gitignore dosyasını kullanmamak
